# Release Notes

### 2026-07-05

- 创建参考锚点时可通过系统文件选择器选择本地参考源文件，仍保留手动输入路径能力。
- 选择 `.txt`、`.md` 或 `.markdown` 文件后，参考源格式会按扩展名自动匹配，减少导入时的手动切换。
- 配置 Embeddings 后，参考锚点导入和重建会为参考材料建立专用向量索引；sqlite-vec 缺失时会显示为可恢复的 embedding 失败状态，已导入材料仍可继续检索。
- 参考复用审计现在会列出 L2 轻度改写中的非槽位改动，例如连接词、标点或小段替换，方便判断候选是否仍在可接受范围内。
- 参考锚定流程现在会持久化接受、拒绝和人工编辑反馈记录，人工编辑只保存哈希而不长期保存编辑正文，为后续材料排序与坏候选回归提供依据。
- 蓝图材料绑定会给已被接受过的参考材料加分，并在评分组成中显示 `accepted_feedback`，让人工选择开始反馈到后续排序。
- 蓝图材料匹配现在可以先返回按 beat 排序的候选而不自动选中；工作区绑定按钮会显式选中每个 beat 的最高分候选后再进入候选草稿生成。
- 候选草稿生成现在允许已批准 `no_reuse_reason` 的过渡 beat 跳过材料链接；普通 beat 仍会因为缺少当前材料链接而被拦截。
- 候选草稿生成会验证每个候选都能追溯到当前选中的材料链接；未批准的 no-reuse 或材料不匹配会被拒绝。
- 候选草稿审计现在会检查 beat 的段落意图、执行模式和候选拒绝规则，纯走位候选不能冒充需要停留/迟疑的段落。
- 候选草稿审计会把 subtext、感官锚点和 source-backed detail 的必需目标纳入检查，缺少指定暗示或细节会要求修正。
- 候选草稿审计现在会拦截 close/limited 叙事距离下的镜头化或上帝视角语言，避免候选越过蓝图 POV 边界。
- 候选草稿审计现在会拦截“POV 角色没有看见/不知道某事，却把那件事告诉读者”的有限视角泄漏，即使该事实本身已在蓝图中获准。
- 候选草稿审计现在会拦截 POV 角色背对/未回头时直接揭示门后、身后隐藏方位的有限视角泄漏。
- 候选草稿审计现在会把未批准的门禁卡、通行证、工牌等准入凭证视为高风险新增事实，避免候选临时发明关键通行物。
- 蓝图生成现在会先归一化章节计划、章节目标、参考锚点和已知/禁止事实组成的上下文包，再计算 `context_hash`，减少等价输入造成的误失效。
- 蓝图生成遇到疑似最终正文段落的章节计划时，会先转成结构化蓝图职责提示，不再把正文原句塞进 beat premise。
- 重复评审未修改的蓝图时会复用已有评审结果，不再产生重复记录，也不会把已批准蓝图退回待批准状态。
- 蓝图评审结果现在包含 `review_version`，审批和后续门禁会拒绝旧评审版本，避免评审规则升级后误用旧结果。
- 蓝图评审现在会拒绝角色状态 before/after 完全相同的 beat，并把问题归入 character-state 缺陷，防止缺少角色状态变化的蓝图继续进入材料绑定和候选草稿生成。
- 蓝图评审现在会拒绝缺少 `character_misbeliefs` 或 `relationship_pressure` 的 beat，让角色状态变化必须具备误信/盲点和关系压力支撑。
- 蓝图评审现在会拒绝“写得更好/更有代入感”这类泛化段落意图，并要求 `paragraph_intention` 写成可检查的具体正文职责。
- 蓝图评审现在会拒绝“正常写/自然展开”这类泛化执行模式，并要求 `execution_mode` 写成可检查的起草操作。
- 蓝图评审现在会拒绝“不好的不要/质量差就拒绝”这类泛化候选拒绝规则，并要求 `candidate_rejection_rule` 写清 action-only、POV 泄漏、缺证据等具体失败条件。
- 蓝图评审现在会拒绝“避免剧本化”这类泛化 anti-screenplay duty，并要求 `anti_screenplay_duty` 写清具体要补足的内心、暗示、细节或非镜头化叙事工作。
- 蓝图评审现在会拒绝“正常叙述/写得有画面感”这类泛化叙事策略，并要求 `narration_strategy` 写清 POV 距离、感官/内心边界和叙事职责。
- 蓝图评审现在会拒绝缺失或“节奏自然流畅/快慢结合”这类泛化节奏策略，并要求 `rhythm_strategy` 写清停顿、推进、转折或释放方式。
- 蓝图评审现在会拦截 final hook 中未由已知事实、beat 场景事实或 POV 已知边界铺垫的高风险身份、证据和过去事件揭示。
- 蓝图评审现在会拦截 beat `scene_facts` 中未由 `known_facts` 或 `slot_plan` 声明槽值支持的高风险身份、证据和过去事件揭示。
- 蓝图评审现在会拦截同一 beat 中 `scene_facts` 与 `viewpoint_forbidden_knowledge` 冲突的事实，防止有限 POV 蓝图把禁止知道的事实当作可用场景事实。
- 蓝图评审现在会返回结构化缺陷，界面可直接显示字段路径、beat、严重度、原因和必须修复项，不再需要解析整段评语来定位问题。
- 蓝图审批会写入独立审批记录，冻结评审 ID、上下文哈希、章节计划哈希、分析合同哈希、评审版本、审批来源和审批时间，后续评审记录变化也不会改写已发生的审批快照。
- 候选草稿审计会继续拦截剧本化的纯对话或纯动作段落，但明确标记为 `short_exchange` 的两句短对话不会再被误判。
- 候选草稿情绪审计现在会把蓝图里的情绪 after-state 也视为已批准证据，避免只写结果状态的短段落被误判为缺少情绪机制。
- 候选草稿情绪审计现在能识别“指尖发紧”与“手指蜷紧”这类等价中文外显证据，不再只依赖逐字命中蓝图短语。
- 参考材料搜索结果现在会返回排序评分组成，让工具和后续界面能解释词面匹配、标签、置信度等因素如何影响结果顺序。
- 参考材料搜索面板现在可直接筛选叙事职责、情绪转变、材料类型、功能、情绪、POV 和技法标签，并在结果卡片上显示评分组成与人工校正状态。
- 参考材料搜索现在支持按叙事职责与情绪转变过滤，蓝图材料检索可以更精确地区分外显证据、感官压力、转场等用途。
- 蓝图材料绑定结果现在会显示每条材料与 beat 的匹配解释，帮助判断材料命中了功能、情绪、POV、正文职责还是历史接受反馈。
- 参考材料标签可由用户手动校正，校正后的 function、emotion、POV、scene 和 technique 标签会标记为用户确认，并直接参与后续材料搜索和蓝图材料绑定。
- 参考材料导入现在能识别更多中文叙事标签，包括外显情绪证据、背对/未回头的有限视角、动作后拍，以及“欲言又止/指节扣紧”这类克制情绪证据；材料搜索和蓝图绑定会更容易找到符合外部证据、有限 POV 或后拍职责的素材。
- 重建参考锚点时会保留文本哈希未变化材料的用户校正标签，源文件前后小幅增删后也不会轻易丢失人工确认结果。

### 2026-07-04

- 新增参考锚定工作区入口，可建立参考素材、搜索素材，并按蓝图评审、素材绑定、候选草稿审核的顺序推进。
- 参考锚定蓝图现在可在工作区内修订事实、POV、情绪证据、段落意图、正文职责和引用查询；保存后会回到待评审状态，必须重新评审并批准后才能继续绑定材料或生成候选段落。
- 蓝图评审和候选草稿审计改为分类展示，可直接查看逻辑、因果、情绪、POV、材料匹配、剧本化风险和必须修复项。
- 材料绑定会显示评分组成，并只允许使用匹配当前蓝图分析合同的有效材料链接生成候选段落。
- 候选段落预览会展示来源材料、节拍、改写等级、变更槽位、非槽位改动和审计风险。
- Agent 可调用参考素材检索、改写审计和蓝图驱动草稿工具，但不会通过该流程直接写入章节正文。
- Novelist 桌面版启动更直接，不再依赖本机 ASP.NET/Kestrel 服务作为产品运行时，安装包启动路径更轻量。
- 封面显示改为通过桌面桥接读取，打包后的应用不再需要 `/covers` HTTP 路由也能显示小说封面。
- 开发模式仍可通过 Vite 地址调试前端，桌面业务调用继续走 Photino bridge。
