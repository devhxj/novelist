**Description:** Phase 13 is the full-product quality gate for Novelist. It must exercise the whole desktop frontend in a real browser, validate bridge behavior with deterministic fixtures, evaluate usability and writing ergonomics, and prove robustness under extreme input sizes and failure modes. This phase must not be reduced to a smoke test or a narrow reference-anchor regression.

The intended standard is: a serious long-form novelist can use the app for large projects without white screens, hidden data loss, confusing flows, brittle controls, or reference-corpus behavior that depends on manual per-novel setup. Reference material must be generated by deterministic segmentation and material extraction from the novel or reference source text passed into the workflow, including large novels, while preserving provenance and auditability.

## Non-Negotiable Scope

- Cover every primary user surface: startup, workspace, bookshelf, novel metadata, chapter list, editor, tabs, save state, chat, agent/tool rendering, search/RAG, reference anchoring, corpus library, material library, characters, locations, timeline, story arcs, preferences, reader perspective, skills, profile, settings, import/export, help, and error recovery.
- Test the app as a product, not only as components: each major workflow must verify visible state, bridge calls, persistence expectations, guardrails, screenshots, and user recovery paths.
- Include usability and ease-of-use evaluation as first-class deliverables: task completion friction, discoverability, confusing labels, excessive clicks, blocked paths, empty-state quality, keyboard ergonomics, and responsive layout issues must be recorded and triaged.
- Include extreme and malformed input coverage: at minimum a 10MB Chinese novel/reference source, very long chapters, many chapters, mixed punctuation, broken encoding-like content, duplicate titles, empty sections, huge material lists, and failed/slow bridge responses.
- Verify automatic corpus/material generation from the supplied novel/reference text. The default flow must not require a user to manually prebuild a corpus, manually split text, or manually bind a novel to a corpus before material extraction, search, blueprint binding, and draft audit can work.
- Preserve all safety boundaries: no automatic `SaveContent` from chat/reference workflows, no arbitrary file reads, no live API key or model dependency, no source-text leakage outside allowed previews, and no chapter mutation before explicit user save.

## Automatic Corpus Generation Requirement

Phase 13 must explicitly test the automatic path below:

```text
passed novel/reference text
  -> deterministic source segmentation
  -> deterministic reference material extraction
  -> stored source hashes and provenance
  -> material search by current story context
  -> blueprint beat material binding
  -> draft candidate audit
```

This is a product requirement, not just a backend detail. Tests must prove that a user can provide a novel or reference source and get usable materials generated from that text without hand-splitting chapters into a separate corpus management workflow. Manual corpus-library operations remain available for inspection, correction, archive, restore, and advanced control, but they cannot be the only path that makes reference anchoring usable.

## Extreme Case Matrix

The Phase 13 suite must include explicit cases for:

- 10MB Chinese novel/reference source.
- Single very long chapter.
- Hundreds or thousands of chapter rows.
- Duplicate novel/chapter titles.
- Empty novel, empty chapter, empty corpus, and empty material search.
- Mixed Chinese/English punctuation, markdown-looking prose, dialogue-heavy prose, and malformed line breaks.
- Slow bridge calls, rejected bridge calls, malformed bridge payloads, and partial streaming events.
- Missing model/provider settings, invalid embedding configuration, and unavailable local model path.
- Restricted/unknown/license-mismatched corpus sources.
- Archived and restored anchors/materials.
- Compact viewport and high-density desktop viewport.
- Production `dist` cache/stale chunk behavior.
